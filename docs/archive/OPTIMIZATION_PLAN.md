# 代码生成器架构优化计划

## 目标
让代码生成器像原生RuoYi一样稳定可靠

## 优化项（按优先级）

### ✅ 1. 表结构同步机制
**问题**：YAML修改后数据库表不会自动更新  
**方案**：部署时检查表结构，提供 `recreate` 选项

### ✅ 2. BaseEntity字段识别
**问题**：create_by/update_by等字段处理不一致  
**方案**：明确识别BaseEntity字段，SQL中正确处理

### ✅ 3. Checkbox简化
**问题**：多个Checkbox时重复声明变量  
**方案**：模仿原生，简化处理逻辑

### ✅ 4. 日期组件统一
**问题**：日期时间格式处理不一致  
**方案**：编辑表单支持datetime类型

### ✅ 5. 字段分类系统
**问题**：字段角色不明确  
**方案**：pk | base | insert | update | query | list

## 实施策略
1. 每个优化单独测试，确保稳定
2. 保持向后兼容，YAML格式不变
3. 新增功能默认关闭，显式启用
